    Custom Domain Implementation Guide¶

    Overview¶

    YES, it is absolutely possible to use custom domains like prs.citylandcondo.com with automated Let's Encrypt SSL certificates, even with GoDaddy hosting (no API access) and office-network-only accessibility.

    This guide covers the complete implementation of custom domains for PRS on-premises deployment, including the proven solution for office-network-only setups with professional SSL certificates.

    Implementation Summary¶

    Solution: HTTP-01 Challenge with Temporary Port Forwarding¶

    This solution works by temporarily exposing port 80 during certificate generation (every ~90 days) while maintaining security for daily operations.

    Key Benefits¶

    • Automated SSL: Let's Encrypt certificates with 90-day auto-renewal
    • GoDaddy Compatible: Works without API access - only requires DNS A record
    • Office Network Only: Domain only accessible from internal network
    • Minimal Maintenance: Quarterly renewal process (5-10 minutes)
    • Professional Appearance: Custom branded domain for users
    • Security Maintained: HTTPS with valid certificates

    Domain Requirements¶

    Prerequisites for Custom Domain¶

    • Static Public IP: Required for SSL certificate validation
    • DNS Control: Administrative access to domain DNS management
    • Router Access: Ability to configure temporary port forwarding
    • Office Network: Internal network configured (192.168.0.0/20)

    Supported Domain Configurations¶

    • GoDaddy Domains: *.citylandcondo.com (primary use case)
    • Other Registrars: Any domain with DNS control
    • Subdomains: prs.your-company.com
    • Internal Domains: prs.company.local (self-signed certificates only)

    Implementation Components¶

    Automated Scripts Available¶

    The PRS deployment includes these custom domain automation scripts:

    • ssl-automation-citylandcondo.sh - Complete SSL certificate automation
    • ssl-renewal-monitor.sh - Daily monitoring and alerting
    • setup-custom-domain.sh - One-click setup wizard

    DNS Configuration¶

    For GoDaddy Domains (Recommended)¶

    Primary DNS A Record¶

    # GoDaddy DNS Management
Record Type: A
Name: prs
Value: [Your office public IP address]
TTL: 1 Hour (3600 seconds)

    Public IP Required

    The DNS A record must point to your office's public IP address, not the internal server IP (192.168.0.100). This is required for SSL certificate validation.

    Optional Internal DNS¶

    # Internal DNS Server (for performance)
prs.citylandcondo.com.     IN  A    192.168.0.100

    For Other Domain Registrars¶

    Standard A Record Configuration¶

    # Primary domain A record
prs.your-company.com.     IN  A    [Office Public IP]

# Subdomain A record
procurement.your-company.com.  IN  A    [Office Public IP]

# WWW alias (optional)
www.prs.your-company.com.      IN  CNAME  prs.your-company.com.

    Quick Implementation¶

    One-Command Setup¶

    For *.citylandcondo.com domains:

    # Run the automated setup script
./setup-custom-domain.sh

    This script handles: 1. Prerequisites check 2. Package installation 3. DNS configuration guidance 4. SSL certificate generation 5. Service configuration 6. Monitoring setup

    Manual Implementation Steps¶

    Step 1: DNS Configuration (One-time)¶

    1. Add A record in GoDaddy: prs → [Office Public IP]
    2. Wait 5-10 minutes for DNS propagation
    3. Verify with: nslookup prs.citylandcondo.com 8.8.8.8

    Step 2: SSL Certificate Generation¶

    1. Enable port 80 forwarding temporarily
    2. Run: ./ssl-automation-citylandcondo.sh
    3. Disable port 80 forwarding after completion

    Step 3: Service Configuration¶

    1. Update environment variables
    2. Restart services: ./deploy-onprem.sh restart
    3. Test access from office devices

    Operational Process¶

    Initial Setup (One-time, ~30 minutes)¶

    1. Configure DNS A record in domain registrar
    2. Run setup script or manual configuration
    3. Follow guided prompts
    4. Test access from office devices

    Quarterly Renewal (~5-10 minutes)¶

    1. Receive automated email alert
    2. Enable port 80 forwarding temporarily
    3. Run: ./ssl-automation-citylandcondo.sh renew
    4. Disable port 80 forwarding
    5. Verify certificate renewal

    Daily Operations (Automated)¶

    • Certificate monitoring at 2:00 AM
    • Health checks and alerts
    • Log rotation and maintenance

    DNS Validation¶

    # Verify DNS resolution
nslookup prs.your-company.com
dig prs.your-company.com A
dig prs.your-company.com AAAA

# Test from different DNS servers
dig @8.8.8.8 prs.your-company.com
dig @1.1.1.1 prs.your-company.com

# Check DNS propagation
for server in 8.8.8.8 1.1.1.1 208.67.222.222; do
    echo "Testing $server:"
    dig @$server prs.your-company.com A +short
done

    SSL Certificate Configuration¶

    Let's Encrypt Certificate¶

    Automated Certificate Generation¶

    # Update domain in SSL automation script
cd /opt/prs-deployment/scripts
cp ssl-automation-citylandcondo.sh ssl-automation-custom.sh

# Edit the script for your domain
sed -i 's/citylandcondo.com/your-company.com/g' ssl-automation-custom.sh
sed -i 's/cityland@citylandcondo.com/admin@your-company.com/g' ssl-automation-custom.sh

# Run certificate generation
./ssl-automation-custom.sh

    Manual Let's Encrypt Setup¶

    # Stop nginx temporarily
docker-compose -f ../02-docker-configuration/docker-compose.onprem.yml stop nginx

# Generate certificate
sudo certbot certonly \
  --standalone \
  --email admin@your-company.com \
  --agree-tos \
  --no-eff-email \
  --domains prs.your-company.com \
  --domains www.prs.your-company.com

# Copy certificates
sudo cp /etc/letsencrypt/live/prs.your-company.com/fullchain.pem \
  /opt/prs-deployment/02-docker-configuration/ssl/certificate.crt

sudo cp /etc/letsencrypt/live/prs.your-company.com/privkey.pem \
  /opt/prs-deployment/02-docker-configuration/ssl/private.key

sudo cp /etc/letsencrypt/live/prs.your-company.com/chain.pem \
  /opt/prs-deployment/02-docker-configuration/ssl/ca-bundle.crt

# Start nginx
docker-compose -f ../02-docker-configuration/docker-compose.onprem.yml start nginx

    Certificate Installation¶

    # Copy your commercial certificate files
sudo cp /path/to/your-domain.crt /opt/prs-deployment/02-docker-configuration/ssl/certificate.crt
sudo cp /path/to/your-domain.key /opt/prs-deployment/02-docker-configuration/ssl/private.key
sudo cp /path/to/ca-bundle.crt /opt/prs-deployment/02-docker-configuration/ssl/ca-bundle.crt

# Set proper permissions
sudo chmod 644 /opt/prs-deployment/02-docker-configuration/ssl/certificate.crt
sudo chmod 600 /opt/prs-deployment/02-docker-configuration/ssl/private.key
sudo chmod 644 /opt/prs-deployment/02-docker-configuration/ssl/ca-bundle.crt

    Certificate Validation¶

    # Verify certificate matches domain
openssl x509 -in /opt/prs-deployment/02-docker-configuration/ssl/certificate.crt -text -noout | grep -A1 "Subject Alternative Name"

# Check certificate chain
openssl verify -CAfile /opt/prs-deployment/02-docker-configuration/ssl/ca-bundle.crt \
  /opt/prs-deployment/02-docker-configuration/ssl/certificate.crt

# Test certificate expiration
openssl x509 -in /opt/prs-deployment/02-docker-configuration/ssl/certificate.crt -noout -dates

    Nginx Configuration¶

    # Custom domain nginx configuration
server {
    listen 80;
    server_name prs.your-company.com www.prs.your-company.com;
    return 301 https://prs.your-company.com$request_uri;
}

server {
    listen 443 ssl http2;
    server_name www.prs.your-company.com;

    ssl_certificate /etc/nginx/ssl/certificate.crt;
    ssl_certificate_key /etc/nginx/ssl/private.key;

    return 301 https://prs.your-company.com$request_uri;
}

server {
    listen 443 ssl http2;
    server_name prs.your-company.com;

    # SSL Configuration
    ssl_certificate /etc/nginx/ssl/certificate.crt;
    ssl_certificate_key /etc/nginx/ssl/private.key;
    ssl_trusted_certificate /etc/nginx/ssl/ca-bundle.crt;

    # SSL Security Settings
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers ECDHE-RSA-AES256-GCM-SHA512:DHE-RSA-AES256-GCM-SHA512:ECDHE-RSA-AES256-GCM-SHA384;
    ssl_prefer_server_ciphers off;
    ssl_session_cache shared:SSL:10m;
    ssl_session_timeout 10m;

    # Security Headers
    add_header Strict-Transport-Security "max-age=31536000; includeSubDomains; preload" always;
    add_header X-Frame-Options DENY always;
    add_header X-Content-Type-Options nosniff always;
    add_header X-XSS-Protection "1; mode=block" always;
    add_header Referrer-Policy "strict-origin-when-cross-origin" always;

    # Custom domain specific headers
    add_header X-Served-By "PRS-OnPrem" always;
    add_header X-Domain "prs.your-company.com" always;

    # Application routes
    location / {
        proxy_pass http://frontend:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header X-Forwarded-Host $host;
    }

    location /api/ {
        proxy_pass http://backend:4000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header X-Forwarded-Host $host;
    }

    # Health check endpoint
    location /health {
        access_log off;
        return 200 "healthy\n";
        add_header Content-Type text/plain;
    }
}

    Multiple Domain Configuration¶

    # Support multiple domains
server {
    listen 443 ssl http2;
    server_name prs.your-company.com procurement.your-company.com prs.yourcompany.local;

    # SSL Configuration (wildcard or multi-domain certificate)
    ssl_certificate /etc/nginx/ssl/certificate.crt;
    ssl_certificate_key /etc/nginx/ssl/private.key;

    # Domain-specific routing
    set $backend_host backend:4000;
    set $frontend_host frontend:3000;

    # Custom routing based on domain
    if ($host = "prs.yourcompany.local") {
        set $backend_host backend-dev:4000;
        set $frontend_host frontend-dev:3000;
    }

    location / {
        proxy_pass http://$frontend_host;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;

    location /api/ {

        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;

    Domain Validation and Testing¶

    Automated Domain Testing¶

    #!/bin/bash
# domain-validation.sh

DOMAIN="prs.your-company.com"
LOG_FILE="/tmp/domain-validation-$(date +%Y%m%d_%H%M%S).log"

echo "Domain Validation Report for $DOMAIN" | tee "$LOG_FILE"
echo "======================================" | tee -a "$LOG_FILE"

# Test DNS resolution
echo "Testing DNS resolution..." | tee -a "$LOG_FILE"
if nslookup "$DOMAIN" > /dev/null 2>&1; then
    echo "✓ DNS resolution successful" | tee -a "$LOG_FILE"
    IP=$(dig +short "$DOMAIN")
    echo "  Resolved IP: $IP" | tee -a "$LOG_FILE"
else
    echo "✗ DNS resolution failed" | tee -a "$LOG_FILE"
fi

# Test HTTP redirect
echo "Testing HTTP to HTTPS redirect..." | tee -a "$LOG_FILE"
HTTP_STATUS=$(curl -s -o /dev/null -w "%{http_code}" "http://$DOMAIN")
if [ "$HTTP_STATUS" = "301" ] || [ "$HTTP_STATUS" = "302" ]; then
    echo "✓ HTTP redirect working ($HTTP_STATUS)" | tee -a "$LOG_FILE"
else
    echo "✗ HTTP redirect not working ($HTTP_STATUS)" | tee -a "$LOG_FILE"
fi

# Test HTTPS access
echo "Testing HTTPS access..." | tee -a "$LOG_FILE"
if curl -f -s "https://$DOMAIN/health" > /dev/null; then
    echo "✓ HTTPS access successful" | tee -a "$LOG_FILE"
else
    echo "✗ HTTPS access failed" | tee -a "$LOG_FILE"
fi

# Test SSL certificate
echo "Testing SSL certificate..." | tee -a "$LOG_FILE"
if openssl s_client -connect "$DOMAIN:443" -servername "$DOMAIN" < /dev/null 2>/dev/null | grep -q "Verify return code: 0"; then
    echo "✓ SSL certificate valid" | tee -a "$LOG_FILE"

    # Get certificate details
    CERT_EXPIRY=$(openssl s_client -connect "$DOMAIN:443" -servername "$DOMAIN" < /dev/null 2>/dev/null | openssl x509 -noout -enddate | cut -d= -f2)
    echo "  Certificate expires: $CERT_EXPIRY" | tee -a "$LOG_FILE"
else
    echo "✗ SSL certificate invalid" | tee -a "$LOG_FILE"
fi

# Test API endpoints
echo "Testing API endpoints..." | tee -a "$LOG_FILE"
if curl -f -s "https://$DOMAIN/api/health" > /dev/null; then
    echo "✓ API endpoints accessible" | tee -a "$LOG_FILE"
else
    echo "✗ API endpoints not accessible" | tee -a "$LOG_FILE"
fi

# Test application loading
echo "Testing application loading..." | tee -a "$LOG_FILE"
RESPONSE_TIME=$(curl -w "%{time_total}" -o /dev/null -s "https://$DOMAIN/")
if (( $(echo "$RESPONSE_TIME < 5.0" | bc -l) )); then
    echo "✓ Application loads in ${RESPONSE_TIME}s" | tee -a "$LOG_FILE"
else
    echo "✗ Application slow to load (${RESPONSE_TIME}s)" | tee -a "$LOG_FILE"
fi

echo "Domain validation completed. Report saved to: $LOG_FILE"

    SSL Certificate Monitoring¶

    #!/bin/bash
# ssl-monitor.sh

DOMAIN="prs.your-company.com"
DAYS_WARNING=30

# Check certificate expiration
CERT_FILE="/opt/prs-deployment/02-docker-configuration/ssl/certificate.crt"
if [ -f "$CERT_FILE" ]; then
    EXPIRY_DATE=$(openssl x509 -in "$CERT_FILE" -noout -enddate | cut -d= -f2)
    EXPIRY_EPOCH=$(date -d "$EXPIRY_DATE" +%s)
    CURRENT_EPOCH=$(date +%s)
    DAYS_UNTIL_EXPIRY=$(( (EXPIRY_EPOCH - CURRENT_EPOCH) / 86400 ))

    if [ $DAYS_UNTIL_EXPIRY -lt $DAYS_WARNING ]; then
        echo "WARNING: SSL certificate for $DOMAIN expires in $DAYS_UNTIL_EXPIRY days"

        # Send notification email
        echo "SSL certificate for $DOMAIN expires in $DAYS_UNTIL_EXPIRY days" | \
        mail -s "SSL Certificate Expiration Warning" admin@your-company.com

        # Auto-renew if Let's Encrypt
        if [ -f "/opt/prs-deployment/scripts/ssl-automation-custom.sh" ]; then
            echo "Attempting automatic renewal..."
            /opt/prs-deployment/scripts/ssl-automation-custom.sh --renew
        fi
    else
        echo "SSL certificate for $DOMAIN is valid for $DAYS_UNTIL_EXPIRY more days"
    fi
else
    echo "ERROR: SSL certificate file not found: $CERT_FILE"
fi

    Domain Migration¶

    #!/bin/bash
# migrate-domain.sh

OLD_DOMAIN="$1"
NEW_DOMAIN="$2"

if [ -z "$OLD_DOMAIN" ] || [ -z "$NEW_DOMAIN" ]; then
    echo "Usage: $0 <old-domain> <new-domain>"
    exit 1
fi

echo "Migrating from $OLD_DOMAIN to $NEW_DOMAIN"

# Update environment files
sed -i "s/$OLD_DOMAIN/$NEW_DOMAIN/g" /opt/prs-deployment/02-docker-configuration/.env

# Update database
docker exec prs-onprem-postgres-timescale psql -U prs_admin -d prs_production -c "
UPDATE system_settings SET value = REPLACE(value, '$OLD_DOMAIN', '$NEW_DOMAIN');
UPDATE email_templates SET content = REPLACE(content, '$OLD_DOMAIN', '$NEW_DOMAIN');
UPDATE notification_settings SET webhook_url = REPLACE(webhook_url, '$OLD_DOMAIN', '$NEW_DOMAIN');
"

# Restart services
docker-compose -f /opt/prs-deployment/02-docker-configuration/docker-compose.onprem.yml restart

echo "Domain migration completed. Please update DNS and SSL certificates."
